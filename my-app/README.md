# Менеджер задач с Redux Toolkit

## Описание проекта
Приложение для управления списком задач на React. Изначально хранение задач  при помощи локального `useState` перенесено в Redux store.


## Установка и запуск
npm install
npm run dev

## Установленные зависимости
Для внедрения Redux добавлены пакеты:
- `@reduxjs/toolkit`
- `react-redux`

## Изменения после внедрения Redux
До внедрения Redux массив задач, логика добавления и переключения статуса находились в компоненте `App.jsx` и управлялись через `useState`.

После внедрения Redux архитектура стала централизованной:
- в `src/store/store.js` создан глобальный store через `configureStore`;
- в `src/store/tasksSlice.js` вынесены состояние задач, reducers, actions и selectors;
- `Provider` подключен в `src/main.jsx`, поэтому Redux store доступен всем компонентам;
- компоненты получают данные через `useSelector` и изменяют состояние через `useDispatch`;
- сохранение задач в `localStorage` теперь выполняется централизованно через подписку на store.

## Какие редьюсеры созданы
В `tasksSlice.js` созданы следующие reducers:
- `addTask` — добавляет новую задачу в начало списка;
- `toggleTaskStatus` — переключает статус задачи между выполненной и активной;
- `deleteTask` — удаляет задачу по `id`.

## Какие селекторы созданы
В `tasksSlice.js` созданы следующие selectors:
- `selectAllTasks` — возвращает все задачи;
- `selectActiveTasks` — возвращает только активные задачи;
- `selectCompletedTasks` — возвращает только выполненные задачи.

## Что изменено в компонентах
- `App.jsx` больше не хранит список задач в локальном состоянии;
- `TaskForm.jsx` отправляет действие `addTask` через `dispatch`;
- `TaskItem.jsx` отправляет действия `toggleTaskStatus` и `deleteTask` через `dispatch`;
- `TaskList.jsx` отображает задачи, полученные после фильтрации;
- `Stats` получает статистику, рассчитанную на основе данных из Redux store.
